# Security Policy

## Supported versions

The latest `main` branch is supported.

## Reporting a vulnerability

Please do not open public issues for security problems.

Instead, use GitHub's private security advisory flow once the repository is published, or contact the maintainer privately and include:

- affected version or commit
- reproduction steps
- impact assessment
- any proof-of-concept details

We will acknowledge the report as soon as possible and work on a fix before public disclosure.
